# Architecture Analysis

Analyze the codebase architecture within scope: `{{SCOPE}}`

## Instructions

1. Use your filesystem and code search tools to explore the codebase
2. Identify the architectural patterns in use
3. Map out the dependency graph between major components
4. Evaluate separation of concerns
5. Check for circular dependencies or tight coupling

If you need clarification on which aspects to focus on, use `request_input` to ask the caller.

When your analysis is complete, call `respond` with your findings. Structure your response with:
- **Architecture Overview**: High-level description of the architecture
- **Component Map**: Key components and their responsibilities
- **Dependencies**: How components interact
- **Strengths**: What's working well
- **Concerns**: Potential issues or technical debt
- **Recommendations**: Suggested improvements
